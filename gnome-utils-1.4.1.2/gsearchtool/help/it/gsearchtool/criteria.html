Criteri di ricerca

Oltre alla semplice ricerca per nome di file descritta in precedenza, il Cerca File di GNOME permette di specificare altri criteri. Per farlo, si dovrà selezionare nella lista a scomparsa (presente a sinistra del pulsante Aggiungi) il tipo di criterio appropriato, e di seguito premere su Aggiungi. Ciò aggiungerà il criterio a quelli già esistenti. Il Cerca File di GNOME cercherà i file che corrispondono a tutti i criteri specificati. Per esempio, la mostra la ricerca di tutti i file con estensione txt modificati negli ultimi 7 giorni e che contengono al loro interno la parola GNOME.

Figura 3. Sessione d'esempio del Cerca File

È possibile rimuovere uno qualsiasi dei criteri aggiunti in precedenza premendo sul pulsante Rimuovi a fianco del criterio. È anche possibile disabilitare temporaneamente un criterio disattivando l'opzione Abilita al suo fianco.

Di seguito un elenco di tutti i possibili tipi di criterio:

  • Nome file cerca per nome di file. Come al solito, è possibile usare nella ricerca i caratteri jolly * e ?. Si ricordi inoltre che i nomi dei file in UNIX sono sensibili alle maiuscole/minuscole.

  • Non cercare nelle sottodirectory se è selezionata questa opzione, il Cerca File di GNOME non cercherà i file nelle sottodirectory della directory di partenza.

  • Proprietario del file cerca secondo il nome dell'utente che possiede il file.

  • Gruppo del file cerca secondo il nome del gruppo proprietario del file.

  • Data/ora dell'ultima modifica cerca i file che sono stati modificati durante gli ultimi N giorni (per essere più precisi, durante le ultime N*24 ore).

  • Non cercare nei filesystem montati se è selezionata questa opzione il Cerca File di GNOME cercherà solamente nelle sottodirectory che sono montate nello stesso dispositivo (disco) della directory di partenza. Per esempio, se la directory di partenza è / ed è montata su /dev/hda (il primo disco fisso), e ha una sottodirectory /home che è montata su /dev/hdb (il secondo disco fisso), allora non sarà fatta la ricerca dentro a /home. Questa opzione può essere utile per escludere dalla ricerca file su supporti removibili (CD-ROM, floppy).

  • File vuoto questo criterio è verificato solo se il file è vuoto. Usato principalmente dagli amministratori di sistema per la manutenzione del sistema.

  • Utente o gruppo non valido cerca i file il cui proprietario o gruppo non è valido nel sistema. Questa opzione può essere utile quando si cercano file appartenenti ad utenti che sono stati rimossi dal sistema. Usata principalmente dagli amministratori di sistema per la manutenzione del sistema.

  • Nomi dei file eccetto corrisponde a tutti i file eccetto quelli il cui nome è qui specificato. Come prima, si possono usare i caratteri jolly. Per esempio, se in questo campo si specifica *~, il Cerca File di GNOME escluderà dalla ricerca tutti i file il cui nome finisce con tilde (~).

  • Ricerca semplice di sottostringa corrisponde a tutti i file che contengono la sottostringa specificata. Poiché questa opzione implica la lettura del contenuto di ogni file, può rallentare notevolmente la ricerca. Nota: in questa opzione non si possono usare caratteri jolly.

  • Ricerca espressione regolare corrisponde a tutti i file che contengono l'espressione regolare specificata. Poiché questa opzione implica la lettura del contenuto di ogni file, può rallentare notevolmente la ricerca.

  • Ricerca espressione regolare estesa corrisponde a tutti i file che contengono l'espressione regolare estesa specificata. Poiché questa opzione implica la lettura del contenuto di ogni file, può rallentare notevolmente la ricerca.

Per quanti non hanno familiarità con la nozione di espressione regolare, diremo semplicemente che è una sintassi estremamente flessibile e potente per il pattern matching (ricerca di occorrenze di modelli di stringhe). Diversamente dalla semplice corrispondenza tramite i caratteri jolly, permette l'utilizzo di operazioni logiche (and, or, not), elenchi di simboli da cercare, controllo che la stringa sia all'inizio della riga e molto altro. Una descrizione dettagliata delle espressioni regolari e delle espressioni regolari estese può essere trovata nella pagina di manuale di grep. I nuovi utenti probabilmente le troveranno troppo complicate da usare, ma per gli utenti con più esperienza, le espressioni regolari possono essere molto utili.
