Introduzione

Il programma bug-buddy è uno strumento che vi guiderà, nel modo più indolore possibile, nel processo di creazione di una notifica di errore («bug report»). In qualsiasi momento si possono modificare le cose e poi inviare, salvare o abbandonare la notifica prodotta. Questo documento descrive la versione 0.9 dell'applicazione bug-buddy.

bug-buddy può essere avviato in molti modi:

  • Aprendo il menu principale di GNOME e si selezioni Programmi->Utilità->bug-buddy .

  • Lanciando bug-buddy al prompt di un terminale tipo gnome-terminal o xterm.

  • Se si sta usando gmc, il file manager di GNOME, per avviarlo è sufficiente cliccare per due volte con il proprio mouse su un file core.

  • Quando un programma di GNOME va in crash, è visualizzata una finestra di dialogo di crash. Se nel sistema è presente bug-buddy, allora una delle opzioni della finestra di dialogo offrirà la possibilità di inviare una notifica di errore. Selezionandola si sarà guidati attraverso un'altra finestra di dialogo che mostra i dettagli del programma che è andato in crash. A questo punto se si continua verrà avviato bug-buddy.

bug-buddy è un programma molto strutturato. In qualsiasi punto si può avanzare oppure tornare indietro per correggere i dettagli precedenti.

Lo scopo di tutto questo è di essere in grado di collezionare tutte le informazioni di cui uno sviluppatore avrà bisogno per riprodurre lo stesso problema e poterlo risolvere. Alcune delle informazioni generate potranno sembrare un po' pedanti, ma non le si cancelli. Torneranno sicuramente utili.
