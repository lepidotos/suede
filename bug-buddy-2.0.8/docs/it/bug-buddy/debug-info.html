Informazioni di debugging

Qui si ha la possibilità di scegliere una di queste tre opzioni.

Ignora questo passo

Se si sta notificando un errore nella documentazione, o uno che non manda in crash il programma, oppure si sta notificando un errore e non si ha un file core associato con esso, si selezioni questa opzione e non ci si curi della sezione seguente di questa guida. Questa è l'opzione predefinita.

Raccolta informazioni sul crash per il debugging

Se bug-buddy è stato avviato perché si è risposto sì a «Notifica un errore» nella finestra di dialogo che dice che qualcosa è andato in crash, si selezioni questa possibilità. Se lo si fa, verranno chieste due cose. Le informazioni richieste dovrebbero già essere state inserite, ma se non lo fossero:

  • Eseguibile: In questa casella si deve inserire il nome del programma come lo si sarebbe scritto alla riga di comando. Per esempio, se Control Center è andato in crash, qui si dovrà inserire gnomecc.

  • ID del processo: In questa casella si deve inserire il process ID (ID del processo) o PID. Probabilmente questa informazione era disponibile nella finestra di dialogo originale dove si è selezionato «Notifica un errore». Se si ha già chiuso questa finestra, si può trovarlo avviando il Gnome System Monitor scegliendo Programmi->Utilità->System Monitor o lanciando gtop da riga di comando. Si troverà il PID nella colonna all'estrema sinistra e il comando nella colonna all'estrema destra (per poter visualizzare questa colonna potrebbe essere necessario scorrere verso destra la finestra).

Raccolta informazioni per il debug da un file core

Se bug-buddy si è avviato perché nel file manager si è cliccato per due volte su un file chiamato core, si selezioni questa possibilità. La si selezioni anche se si ha un file core che si sa essere stato creato in conseguenza all'errore che si sta notificando. Verrà visualizzata un'altra casella che mostrerà la posizione del file core.
