Raccolta informazioni di debugging

Questa pagina potrebbe intimidire un po'. La si vedrà se nella schermata precedente si è selezionato «Raccolta informazioni sul crash per il debugging» oppure «Raccolta informazioni per il debug da un file core».

Quando l'applicazione è andata in crash, è stata congelata una copia di tutti dettagli su quel che il programma stava facendo. Questi sono stati scaricati in un file core oppure conservati sul posto. Quel che bug-buddy sta facendo è di scorrere tutte queste informazioni per scoprire quali passi il programma ha fatto per arrivare al crash. I programmatori riescono a seguire questo output piuttosto criptico e capire dove qualcosa ha cominciato ad andar storto. Questo output sarà allegato automaticamente alla propria notifica d'errore. Non lo si modifichi a meno che non si sappia esattamente quel che si sta facendo.

Talvolta bug-buddy non sarà in grado di generare le informazioni e dirà che non riesce a riconoscere il file core. Se così fosse, si prosegua al passo successivo. Non si invii il file core come soluzione alternativa: bug-buddy vuole solo l'output.
