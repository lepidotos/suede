Component

There are three sections to fill in here:

Component

The product selected will have one or more components to it. If there is only one component (typically a general component), click on it. If there are several components, select the most likely-looking one.

Version

This is the version number for the program or package you're reporting a bug in. It may already be filled in. If not, find out through the package's About box and fill it in here.

Severity

This is the severity of the bug, as you feel it. There are several choices in decreasing order:

  1. An enhancement bug is a request for enhancement rather than a bug describing a problem. "It would be nice if.."

  2. A trivial bug is precisely that: trivial. It is typically a cosmetic problem such as misspelt words or misaligned text.

  3. A minor bug involves either a minor loss of function, or a larger problem but one which has an easy workaround.

  4. A normal bug is the typical sort of bug. This is the default, and probably won't need increasing.

  5. A major bug involves a major loss of function.

  6. A critical bug is one which involves a crash, a severe memory leak, or loss of data.

  7. A blocker bug is rare and usually assigned as a blocker by a developer. It is a bug which stops other things being fixed and which must be fixed before other bugs can be closed.

When all three sections are filled in, the Next will become un-greyed out, and you can continue.
